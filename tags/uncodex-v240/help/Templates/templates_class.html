HTML Template class.html

You can use the same replacements as in packages_list_entry.html. These replacements can only be used in class.html, classes_list_entry.html and package_class_table_entry.html.
replacement replaced with 
%class_link%
 Relative location the class detail page 
%class_name%
 the name of the class 
%class_modifiers%
 the class modifiers 
%class_parent%
 the parent class, including link to the parent class page 
%class_parent_plain%
 just the parent class name 
%class_parent_tree%
 a ASCII tree displaying the inheritence tree up to the root 
%class_filename%
 the filename of the class's source 
%class_fileage%
 the last file change of the class's source, format is defined in the template.ini 
%class_defaultproperties%
 the default properties defined in the class source 
%class_comment%
 the description of the class, if present 
%class_children%
 direct sub classes of this class, including links to their pages. The title from the template.ini is added to this 
%class_previous%
 location to the previous class in this package 
%class_source_previous%
 location to the source code HTML file of the next class in this package 
%class_source_next%
 location to the source code HTML file of the previous class in this package

Block templates

Block templates are wild card replacements, the template file used is the the complete replacement string.
Example:
Replacement: %class_block_const_summary%
Template used: class_block_const_summary.html

Block replacements are replaced with an empty string when the current class does not contain any of the elements those block covers (e.g. for %class_block_const*% it checks the class's constants)
Parent classes will also be checked if they contain elements. The number of parent classes checked is controlled in the template.ini file by the "MaxInherit" value. This can be overwritten per template setting a value for this template in the [InheritCheck] block of the template.ini
replacement replaced with 
%class_block_children*%
 Direct subclasses of this class 
%class_block_const*%
 Constants 
%class_block_variable*%
 Variables 
%class_block_enum*%
 Enumerations 
%class_block_struct*%
 Structures 
%class_block_function*%
 Functions, including: events, operators, delegates 
%class_block_state*%
 States 
%class_block_defaultproperties*%
 Defaultproperties, only for this class, parent classes are not checked

Block entry templates

These replacements behave just like the block templates except for the inheritence check, this isn't used.
The entry templates are used to fill in the actual data,
replacement replaced with 
%class_entry_const*%
 Class constants, entries constructed from class_entry_const*.html 
%class_entry_variable*%
 Class variable, entries constructed from class_entry_variable*.html 
%class_entry_enum*%
 Class enumerations, entries constructed from class_entry_enum*.html 
%class_entry_struct*%
 Class structs, entries constructed from class_entry_struct*.html 
%class_entry_function*%
 Class functions, entries constructed from class_entry_function*.html 
%class_entry_state*%
 Class states, entries constructed from class_entry_state*.html

Inherited elements templates

All inherited elements templates are cunstructed from the same template file: class_entry_inherited.html
The depth is constrolled from the template.ini file with the "MaxInherit" value.
