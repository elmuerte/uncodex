Configuration file

You can use the -config commandline argument to load a diffirent configuration file. Below is the definition of the usefull config parameters.

[Config]

name type description 
AnalyseModified
 0 or 1 If set to 1 UnCodeX is search for modified files and analyse them 
ClassPropertiesWindow
 0 or 1 Open the class properties as a window 
ClientCmd
 commandline Commandline to execute to join the game server 
CompilerCmd
 commandline Commandline to execute to compile the selected class/package 
CPP
 commandline Path to the Comment PreProcessor to use, leave empty to use the Template default or 'none' to use none at all 
DefaultInheritanceDepth
 integer Default inheritance depth of the class properties 
FullTextSearchRegExp
 0 or 1 Full text search and class body searches default to regular expression when set to 1 
HHCPath
 path Path to the HTML Help compiler (hhc.exe) 
HTMLHelpFile
 file path Output filename for the HTML Help compiler 
HTMLOutputDir
 path Path to store the output HTML files in 
TargetExt=
 The target extention of the output files. This overrides the template's default setting. 
GZCompress
 -1/0/1 Either 0 (disabled), 1 (enabled) or -1 (use the template's setting). When enabled all HTML files will be compressed with gzip (all except index.html). If the target extion doesn't end on .gz it will be appended. This option is usefull when there's not a lot of diskspace available. Most browsers support gzip compressed files when they are opened online, however it doesn't always work when opened from disk in MS Windows. Default: -1 
InlineSearchTimeout
 integer Number of seconds before the inline search times out 
LoadOutputModules
 0 or 1 Search and load custom output modules when set to 1 
OpenResultCmd
 commandline commandline to use when opening search results, if you leave this blank the default application is used 
ServerCmd
 commandline Commandline to execute when a server needs to be started 
ServerPrio
 0 to 3 Priority of the server process, 0 being the lowest and 3 being the highest 
StateFile
 filename Filename of the state file, relative to the configuration file 
TabsToSpaces
 integer Number to spaces to convert tabs with in the HTML output, if 0 use the template's default setting and if -1 don't convert at all. 
TemplateDir
 path Directory which contains the templates to use for the HTML output

[HotKeys]

This is where the horkeys are set for each menu/toolbar item.

[IgnorePackages]

Package
 Directories matching these names will be ignores if they are identified as a package.

name type description

ABRight

ABWidth
 integer Width of the tool application window 
ExpandObject

 name Font to use in the log window 
Log.Font.Size

LogHeight
 integer Height of the log window 
MenuBar
 0 or 1 Show the menu bar if set to 1 
MinimizeOnClose
 0 or 1 Minimize UnCodeX when the user closes the window, unless it's closed via "Tree -> Quit" 
PackageTree
 0 or 1 Show the package tree 
PackageTreeWidth
 integer Width of the package tree 
SavePosition
 0 or 1 Save the position of UnCodeX 
SaveSize
 0 or 1 Save the size of UnCodeX 
StayOnTop
 0 or 1 If set to 1 the main window willstay on top of all other windows 
Toolbar
 0 or 1 Show the toolbar 
Top
 integer Position to the top of the main window 
Top
 integer Position to the top of the main window 
Tree.Color
 color Background color of both trees 
Tree.Font.Color
 color Text color 
Tree.Font.Name
 name Font to use in both trees 
Tree.Font.Size
 integer Font size 
Width
 integer Width of the main window 
Source.Color
 color background color of the source snoop window 
Source.CF1
 color source snoop syntax highlight: strings/ints/floats 
Source.CF2
 color source snoop syntax highlight: comments 
Source.CF3
 color source snoop syntax highlight: macros 
Source.CF4
 color source snoop syntax highlight: names 
Source.CF5
 color source snoop syntax highlight: types 
Source.Font.Name
 name font to use 
Source.Font.Size
 integer font size 
Source.Tabs
 integer Tab size 
SourceSnoop
 1 or 0 SourceSnoop is visible when set to 1 
SourceSnoopWidth
 integer Size of the source snoop window

[PackagePriority]

Package
 The priority of the package, the first item has the highest priority. 
Tag
 A tagged package, this package and containing classes will get a diffirent icon in the tree listing. The order doesn't matter

[SourcePaths]

Path
 Paths to scan for packages
