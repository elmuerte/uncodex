Changes since 151:

  • "Class changed since last time" log entries are now selectable to focus the changed class
  • Added license window
  • Various little cosmetic fixes

Changes since 150:

  • structs back in html output
  • draw fix for the source snoop
  • fix in tree state saving
  • added defaultproperties to save state
  • added "class" and "package" names to tree popup menu

Changes since 109:

  • comments are now saved in the state file
  • structures are now analysed (Warning: HTML output doesn't do anything with this)
  • properties window will now focus the property when source snoop is active
  • updated wikifier
  • changed license from GPL version 2 to the LGPL

Changes since 108:

  • Fixed the no comment preprocessor error
  • Templates can set the tree drawing items, you can change the default ASCII art tree to a graphic tree (Warning: may get very slow to display)
  • Fixed a bug component sizes on the initial display

Changes since 107:

  • New commandline option: `-fts` to pop-up the full text search on launch
  • Fixed the full text search on an empty search
  • Backspace in the inline search now searches from the top
  • Added DDE support to send commands to UnCodeX, serivce: UNCODEX, topic: EXEC, item: CMD. The content is the same as the commandline argumens

Changes since 106:

  • Added tab completion to the inline search

Changes since 105:

  • "ok" button in the search form is disabled when the input is empty
  • the last query is as default value of the search form
  • the inline search ignores characters when they don't match class
  • added new "open class" place holders:
    • %searchquery%, will be replaced with the inline search query, or when that is empty the last class search query
    • %inlinesearch%, will always be replaced with the inline search query
    • %classsearch%, will always be replaced with the last class search query

Changes since 104:

  • Fixed empty search query in full text end status reply
  • Fixed search hilighting in the source snoop
  • Added key to press to abort in status messages when running

Changes since 103:

  • Keywords are now loaded from the file 'keywords.list' (when available), this way it's easier to add new keywords
  • Fixed the initial `-find`

Changes since 102:

  • Added inline search for both trees, just start typing the name of the class and it will look it up, you can use ESC to cancel the search, Backspace to remove a letter and F3 to find the next class with that name. The inline search will be cancelled after 5 seconds (can be changed).
  • Fixed the broken source snoop window when UnCodeX started with the source snoop window too small
  • Added support for customer Comment PreProcessors (for the HTML output)

Changes since 101:

  • Added the first comment command: /** @ignore */ the following class/var/etc. will be completely ignored in the HTML output

Changes since 100:

  • New HTML output option: TabsToSpaces, converts tab chars to spaces (when set to a non-zero value). Default for the included templates is 4

Changes since 062:

  • Fixed error when the parser encountered an unterminated string or name. The rest of the line is now accepted as string/name. (Note: that unterminated names and strings don't compile in ucc make)

Changes since 061:

  • Type identifier with a class name preceding it (var ComeClass.MyStruct i;) are now parsed correctly.
  • Types with a classname prefix are now also linked
  • Added a button to move a package from the priority list to the ignore list.
  • Added a new templace replacement %has_comment_?% which will be replaced by the title "HasCommentValue" when the function/var/.. has a comment

Changes since 060:

  • The initial comments of a class are now used as class comment, the // prefix is not removed from those comments.
  • HTML output now processes the class comments when they are prefixed with a // the possible horizontal rules are stripped. For this to work you have to make sure that the reverse of the line is the same as the original. Note that using three or more / breaks this method.
