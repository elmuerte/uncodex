version history

Changes since 060:

  • The initial comments of a class are now used as class comment, the // prefix is not removed from those comments.
  • HTML output now processes the class comments when they are prefixed with a // the possible horizontal rules are stripped. For this to work you have to make sure that the reverse of the line is the same as the original. Note that using three or more / breaks this method.

Changes since 058:

  • Added category groups to the variables, because of this the previous state files are incompatible again
  • Improved the UnrealWiki class wikifier, it now adds everything also comments (you have to anaylse the class first for the comments to be included)

Changes since 057:

  • cleaned up part of the source code, nothing that would affect the actual program
  • Added support of output modules for selected classes, this however the way output modules work, so you have to update you source
  • Added an experimental class wikifier for UnrealWiki
  • The HTML Help title can be set in the settings window
  • The default inheritance depth for the class properties can be set
  • The state file format may change in the next version to include some more information about variable declarations

Changes since 056:

  • function name + arguments is now copied in the properties window
  • started with the custom output module support. check the out_sample directory for more information

Changes since 055:

  • Mainly documentation updates
  • Improved the hints a bit

Changes since 054:

  • Fixed tree collapsing/expanding
  • Added more descriptive hints to the class properties window (needs to be improved)

Changes since 053:

  • Fixed occasional uppercase chars in the HTML output filenames
  • Added support for package comments. To use them create a file called 'uncodex.ini' in the package directory and add the following text: 
    [package_description]
some descriptive text about this packages
can cover multiple lines, you don't need to use normal a normal convention used in .ini.
That is, you don't have to use name=value pairs for this part
  • Updated HTML templates to include class and package comments on the overview pages
  • Fixed the analyser, function arguments are cleaner now (no more useless spaces)
  • Added links to declarations of the function arguments

Changes since 052:

  • Improved the class properties window display
  • State is only saved when the tree is really changed, not everytime when searched for modified classes

Changes since 051:

  • Improved the class properties window positioning
  • Fixed the -hide commandline option

Changes since 050:

  • Previous state files are incompatible, this because I had to change the format for a new feature
  • New command: analyse modified class. This will re-analyse class files changed since the last version
  • Auto analyse modified classes
  • Fixed the "Run server" command
  • Fixed a couple of memory leaks in the class analyser

Changes since 049:

  • started on the help file
