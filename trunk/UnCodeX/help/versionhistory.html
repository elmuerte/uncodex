version history

Changes since 216

  • Fixed issue when recreating classtree\analyse all and property inspector was hidden
  • Added more exception catching during package scanning
  • More memory leak fixes
  • Improved conditional compiling for the GUI. Define DEBUG_BUILD for a debug build with the right settings.

Changes since 215

  • Debug compiles (unless I forget to enable it) are now compiled with a memory leak detection ( http://v.mahon.free.fr/pro/freeware/memcheck ). To enable the memory leak detection start UnCodeX with -findmemleak. This will produce a file called UnCodeX_MemCheck.log when UnCodeX is closed. If that file contains information please email it to me.

Changes since 214

  • Fixed a serious bug in both parsers, unterminated block comments where fatal
  • Added more guards to catch errors

Changes since 213

  • bug fix: tags where not copied for multiple variable definitions in unrealscript.
  • #include macro's are now processed and the information is included in the class reference. However, line numbers of included entities are incorrect.
  • Fixed a bug in the HTML otuput comment parsing, some links where not converted correctly.
  • Added new function to UClasses: function declaration: string; This returns a valid unrealscript declaration of that function/variable/etc. Note: this may not be exact as the source declaration.
  • Variables with an empty tag (var()) now get their tag set to the class name.
  • source snoop component updates
  • more #include updates for the GUI
  • Added states to the property list

Changes since 212

  • Initial macro processor code, supports #define, #if, #else and #endif
    #if expression evaluation is limited.
  • Class tree analisation is now tree based instead of alphabetica (required for #defines).
  • Added #ifdef macro support
  • Implemented #if expression evaluator
  • "create subclass" now checks if the class already exists in the current package
  • Added support for subdirectories in packages (Tribes:V feature).
  • Improved class tree constructions, it should now create a correct class tree for all duplicates (was an issue with Tribes:V).
  • Clear command stack for batch processes when a new batch is started.
  • Fixed bug in HTML output for Package.Class types besides the Class.Type types
  • Added %if:something% and %endif% statements to the HTML template, the something differs per template. The if statemens can be nested.

Changes since 211

  • Fixed opening op package directories via the tree context menu
  • Added support for UnrealEngine2 style variable descriptions. These have a lower priority than the description in the comment because the comment has more freedom of expression.
  • Added a hook for macro processing in the code anaylzing parser (Tribes:V support requirement)
  • Added support for interface classes (Tribes:V support requirement)
  • Added support for the FreePascal compiler for ucxcu

  • Fixed issues with the property inspectors after a tree rebuild or analyse
  • Changed the way output modules are loaded, no more 2 different module version. UClasses revision number is used to check for valid modules.
