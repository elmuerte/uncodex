Commandline options

UnCodeX accepts the following commandline options: 
-batch command ...
 Start UnCodeX in batch mode, command can be one or more of the following seperated by spaces: 
rebuild
 rebuilds the class tree 
analyse
 analyse all classes in the tree 
analysemodified
 analyse only modified classes in the tree 
findnew
 find new classes in the current packages, it's best to call this before an "analysemodified" call 
createhtml
 create all HTML file 
htmlhelp
 compile the MS HTML Help file 
close
 close UnCodeX 
orhpanstop
 Stop executing the batch command when a orphan class has been detected 
ext:<name>
 call the custom output module out_name.dll 
--
 end of batch command input, use this if you want to add more command line options after the batch sequence 
-config filename
 Use filename as config file 
-find class name
 Find and select class name in the class tree 
-handle window handle
 The window with window handle will receive status messages from UnCodeX, read more about this option in the developers section 
-help
 Displays a list of accepted commandline options 
-hide
 don't show the main window of uncodex, note that this will prevent you from shutting it down when you don't use "close" the batch command 
-nosplash
 Prevents the splash screen from showing up 
-open class name
 Find and open the source file of class name 
-reuse
 Re use a previous UnCodeX instance, this option depends on the configuration file used 
-tags class name
 Find and open the details of class name 
-fts
 Show the Full Text Search dialog on startup

CTAGS output module commandline options

The following commandline options are accepted by the CTAGS output module: 
--ct-out filename
 The output filename, if no filename is specified you will be prompted for the output file. 
--ct-include spec
 Types to include in the output file (case sensitive). By default all types are included. The following types are available: 
C
 classes 
v
 variables 
c
 constants 
s
 structs 
e
 enumerations 
f
 functions 
d
 delegates 
--ct-offset spec
 Use numeric offset for these packages (case sensitive). by default numeric line offsets are only used for tagged packages. The following types are available: 
t
 tagged packages 
u
 untagged packages 
--ct-untagged
 select untagged packages only 
--ct-tagged
 select tagged packages only 
--ct-packages list
 comma seperated list of packages to select for output.
