Program settings

Source paths

Here you can add all the paths to directories containing UScript packages. These paths will be scanned for possible packages. Usually you just point this to the directory of your Unreal Engine game.

Package Priority

This is the priority listing of the packages, the most important packages must be at the top, this is usualy "Core". You can simply import the priority listing from the game file of an Unreal Engine game using the "..." button. Warning! importing from an .ini file will clear the current list.
New packages will automatically be added at the bottom of the list when they are found. To check if the package priority is set correctly you can use the "Find orphans" command in the main window. If the package priority is set correctly you should have no orhpan files.
You can "tag" a package by checking the box before the package name. A tagged package and all it's containing classes will get a diffirent icon in the Package and Classes tree listing. This way you can distinguish the original packages and classes from your own.

Ignore Packages

Here you can add names of packages that should be ignored when you rebuild the tree.

HTML Output

Output directory

This is the directory where all HTML files will be saved to

Templates directory

This is the directory containing the HTML templates to be used when creating the HTML output.

MS HTML Help

HTML Help Workshop

This is the path to the directory of the HTML Help Workshop. UnCodeX searches for HHC.exe in that directory.

Output file

The file the HTML Help Workshop should create.

Compile

Here you can define the command to execute when you select "Compile class". If the path name contains spaces you must use quotes around it.

Wrong

c:\program files\umake\umake.exe %classpath%

Correct

"c:\program files\umake\umake.exe" %classpath%

You can use the following placeholders in the commandline: 
%classname%
The name of the class 
%classfile%
The filename of the class, this does not include the path 
%classpath%
The full path to the class file, including the filename itself 
%packagename%
The name of the package of the class 
%packagepath%
The path of the package

Game server

Server commandline

The commandline to execute when you want to start a server. Just as with the Compiler commandline you have to place quotes around parts that contain spaces.

Server priority

Here you can set the base priority of the server process executed by the commandline. It's adviced that you set it to at least "High" when you want to join the server running on your local system. Other wise the game will appear to lag.
Note: that if you specified a batch file as commandline this will increase the priority of that batch file, not the actual server.

Client commandline

The command to execute when you want to join the server.

Full Text Search

Open result command

The command to execute when you double click the result of a full text search, or when you double click a property in the details window.
You can use the same placeholders as with the compile command with these additions: 
%resultline%
The line number of the result 
%resultpos%
The position in the result line

Default to regualr expression

With this checked regular expressions will be used by default in full text and class body searches.

Layout

Here you can change the color and font to be used in the tree list and the log window.

Expand "Object" on start up

Expand the first item of the class tree (usualy "Object") when UnCodeX starts.

Program Options

State file

The filename the current tree state will be saved to. This name is relative to the configuration file used. So it should not contain a path, just a filename.

Minimize on close

Minimize UnCodeX when the users pressed the "x" in the window. To really close the application select "Quit" from the menu.

Hot keys

Here you can change the default hot key settings. Select a command from the list, enter a new hot key and press "Set" to update.
