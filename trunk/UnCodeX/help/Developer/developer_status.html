Receiving status messages

UnCodeX gives you the ability to receive it's status and log messages when you execute it in the background. In order to receive the status messages you have to report the window handle of the receiving application to UnCodeX. Only one window can receive the messages at the time. To report the window handle you have to start UnCodeX with the -handle commandline argument.

When you have reported the window handle to UnCodeX it will start sending messages to that handle. UnCodeX will use the Window Message: WM_COPYDATA to send the following structure to the receiving window: 

TCodeXStatus = packed record
  Msg:      string[255];
  mType:    TCodeXStatusType;
  Progress: byte;
end;

TCodeXStatusType = (cxstLog, cxstStatus);
When mType is cxstStatus the Progress may contain a progress percentage of the total operation.
