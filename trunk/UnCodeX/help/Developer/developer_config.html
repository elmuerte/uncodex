config

Configuration file

You can use the -config commandline argument to load a diffirent configuration file. Below is the definition of the usefull config parameters.

Main configuration

The main configuration section, this is the most usefull part of the configuration. It's used by both GUI and commandline version of UnCodeX.

[Include]

Files to be included while loading the configuration. Note: the settings introduced by the include files will be saved to the actual config file the next time it's saved.
Name Type Description
Pre string array These files will be included before the rest of the configuration is loaded. Use this to set global defaults.
Default: UNIX Commandline Only:
Pre=/etc/uncodexrc
Pre=~.uncodexrc
Post string array These files will be included after the rest of the configuration is loaded. Use this to force certain settngs.

[Packages]

Package configuration settings.
Name Type Description
EditPackages string array Known UnrealScript packages. The order is important and can be used to resolve class mismatch errors. The higher the package in the list the more important the package will be. This is only relevant in case of class name collisions. The class in the package with the highest priority will be used as parent. Note: you should try to avoid creating duplicate class names.
Tag string array These packages (and classes contained within it) will be tagged. This is only used in the GUI to give the tagged packages and classes a different icon.
IgnorePackage string array These packages will completely be ignored by the rest of the system.

[Sources]

Configuration for the UnrealScript source code.
Name Type Description
Path string array Locations to directories containing UnrealScript packages.

[HTMLOutput]

Configuration for the HTML Documentation generations.
Name Type Description
OutputDir string Target directory for the generated documents.
Default: 
OutputDir={appdir}\UnCodeX-Output
TemplateDir string Directory to the template to use for creating the documentation
Default: 
OutputDir={appdir}\Templates\DocStyle2\
CreateSource tribool Forces of syntax highlighted source code output in the generated documentation. You shouldn't change this, it's better to create a modified template because the source code might or might not be linked.
Default: 
CreateSource=Maybe
TabsToSpaces integer Defines the number of spaces to use for tabs. Set to -1 to omit conversion of tabs to spaces. 0 will use the template's setting.
Default: 
TabsToSpaces=0
TargetExtention string Sets the target extention of the created files. When empty it uses the template's settings.
Default: 
TargetExtention=
CommentPreProcessor string Path to a comment preprocessor. When set this program will be called to processes the comments for packages, classes, functions, etc. Otherwise the builtin comment processor is used.
Default: 
CommentPreProcessor=
DefaultTitle string Set's the default title. When empty it will use the template's setting for DefaultTitle, if that is empty to it will use 'UnCodeX'. The value of this setting can be used by templates. It's usefull to change this to a descriptive title like: DefaultTitle=UT2004 UnrealScript
Default: 
DefaultTitle=
GZCompress tribool Forces GZCompression of the output. When set to Maybe it will use the template's setting. It's not recommended to set it to False. When set to True all generates files (with the exception of the index file) will be compressed with the gzip algorithm. This will reduce the size of the output to 33% (on an average basis). The suffic .gz will automatically be added to the TargetExtention, unless it's already present.
Default: 
GZCompress=Maybe

[HTMLHelp]

Settings to create a MS HTML Help document from the HTML output.
Name Type Description
Compiler string Path to the HTML Help compiler.
Default: 
OutputDir=
OutputFile string Destination file
Default: 
OutputDir={appdir}\UnCodeX-UnrealScript.chm
Title string Title to use for the HTML Help file, this will be displayed as caption for the help window. When empty it will default to UnCodeX/
Default: 
Title=
